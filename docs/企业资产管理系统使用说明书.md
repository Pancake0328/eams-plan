## 2. 系统概述

### 2.1 系统简介

企业资产管理系统（EAMS）是一套面向企业的全生命周期资产管理信息化平台。系统覆盖资产从采购申请、入库登记、日常使用、调拨流转、维修报废等全流程管理，帮助企业建立规范的资产管理体系，提升资产利用率，降低资产管理成本。

### 2.2 功能模块介绍

系统共包含以下五大功能模块：

#### 2.2.1 系统管理模块

提供系统基础配置能力，包括：
- **用户管理**：维护系统用户账号、设置用户状态、重置密码、绑定角色和所属部门。
- **角色管理**：定义系统角色，灵活配置每个角色可访问的菜单和可操作的功能按钮。
- **部门管理**：以树形结构维护企业组织架构，支持多级部门层次。
- **权限管理**：统一管理系统中所有菜单项和功能按钮权限，支持细粒度权限控制。

#### 2.2.2 资产管理模块

实现资产全台账管理，包括：
- **资产分类管理**：以树形结构定义资产分类，支持多级分类体系（如：IT设备 → 计算机 → 笔记本电脑）。
- **资产信息管理**：完整记录资产基本信息，支持多条件查询、批量操作、Excel导入导出。
- **资产流转管理**：支持入库、分配/调拨、归还、报废、送修等操作，并自动记录流转历史。
- **流转记录查询**：查看所有资产的历史操作记录，追溯资产流转轨迹。

#### 2.2.3 生命周期管理模块

- 对单个资产进行全生命周期追踪，以时间轴形式展示从入库到报废的完整历史。

#### 2.2.4 盘点管理模块

- 支持新建全面盘点、抽样盘点、专项盘点三种类型的盘点计划。
- 支持按部门或资产分类划定盘点范围，设定盘点时间区间。
- 盘点执行过程中逐一确认资产状态，完成后自动汇总盘点结果。

#### 2.2.5 采购管理模块

- 支持发起包含多种资产明细的采购申请单。
- 多级审批流程，审批通过后执行采购入库，将资产自动录入资产台账。

### 2.3 系统特点

1. **全流程管理**：覆盖资产采购、入库、使用、维修、报废全生命周期。
2. **权限精细化**：基于角色的访问控制（RBAC），支持菜单级和按钮级权限控制。
3. **操作可追溯**：所有资产操作均有日志记录，操作历史可随时查询。
4. **数据可视化**：仪表盘提供多维度图表，直观呈现资产分布和状态统计。
5. **数据导入导出**：支持Excel批量导入资产，支持资产数据导出，便于离线分析。
6. **响应式界面**：基于现代前端框架构建，界面简洁美观，操作体验流畅。

## 3. 系统安装与配置

### 3.1 环境要求

#### 3.1.1 服务端环境

| 组件        | 最低版本要求 | 推荐版本  |
| ----------- | ------------ | --------- |
| JDK         | 11           | JDK 17    |
| MySQL       | 5.7          | 8.0+      |
| Redis       | 5.0          | 6.0+      |
| Maven       | 3.6          | 3.8+      |
| 操作系统    | CentOS 7 / Ubuntu 18.04 / Windows Server 2016 | —  |

#### 3.1.2 前端环境

| 组件        | 最低版本要求 |
| ----------- | ------------ |
| Node.js     | 14.x         |
| npm         | 6.x          |

#### 3.1.3 客户端浏览器要求

| 浏览器       | 最低版本 |
| ------------ | -------- |
| Google Chrome | 90+     |
| Mozilla Firefox | 88+  |
| Microsoft Edge | 90+   |

> ⚠️ **注意**：不支持 Internet Explorer 浏览器，请勿使用 IE 访问本系统。

### 3.2 数据库初始化

1. 登录 MySQL 数据库服务器，创建数据库：
   ```sql
   CREATE DATABASE eams CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
   ```
2. 创建数据库用户并授权：
   ```sql
   CREATE USER 'eams_user'@'%' IDENTIFIED BY 'your_password';
   GRANT ALL PRIVILEGES ON eams.* TO 'eams_user'@'%';
   FLUSH PRIVILEGES;
   ```
3. 执行项目 `sql/` 目录下的初始化脚本：
   ```bash
   mysql -u eams_user -p eams < sql/init.sql
   ```

### 3.3 后端服务部署

1. 修改配置文件 `src/main/resources/application.yml`，配置数据库连接信息：
   ```yaml
   spring:
     datasource:
       url: jdbc:mysql://localhost:3306/eams?useUnicode=true&characterEncoding=utf8
       username: eams_user
       password: your_password
   ```
2. 在项目根目录执行 Maven 构建：
   ```bash
   mvn clean package -DskipTests
   ```
3. 启动后端服务：
   ```bash
   java -jar target/eams-*.jar
   ```
4. 默认服务端口为 `8080`，启动成功后控制台输出 `Started Application` 即表示启动成功。

### 3.4 前端服务部署

1. 进入前端目录：
   ```bash
   cd frontend
   ```
2. 安装依赖：
   ```bash
   npm install
   ```
3. 开发模式启动（本地调试）：
   ```bash
   npm run dev
   ```
4. 生产模式构建：
   ```bash
   npm run build
   ```
   构建产物位于 `dist/` 目录，将其部署到 Nginx 或其他 Web 服务器即可。

### 3.5 Nginx 参考配置（生产环境）

```nginx
server {
    listen 80;
    server_name your-domain.com;

    root /var/www/eams/dist;
    index index.html;

    location / {
        try_files $uri $uri/ /index.html;
    }

    location /api/ {
        proxy_pass http://127.0.0.1:8080/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
}
```

### 3.6 系统访问地址

- **前端访问地址**：`http://localhost:3000`（开发模式）或部署后的域名/IP
- **后端 API 地址**：`http://localhost:8080`

## 4. 系统登录与注销

### 4.1 系统登录

【截图：登录页面截图】

**操作步骤：**

1. 打开浏览器（推荐 Chrome 90+），在地址栏输入系统访问地址 `http://localhost:3000`，按回车键访问。
2. 系统显示登录页面，页面中央呈现登录表单。
3. 在 **用户名** 输入框中输入账号，初始管理员账号为 `admin`。
4. 在 **密码** 输入框中输入密码，初始管理员密码为 `123456`。
5. 点击 **登录** 按钮，系统验证身份信息。
6. 验证通过后，系统自动跳转至仪表盘首页。

> ⚠️ **注意**：首次登录后请立即修改默认密码，以保障系统安全。
>
> ⚠️ **注意**：连续输错密码超过5次，账号将被临时锁定，请联系系统管理员解锁。

> 💡 **提示**：如需记住登录状态，可勾选"记住我"选项，系统将在一定时间内免密码访问。

### 4.2 修改登录密码

1. 登录成功后，点击页面右上角的用户头像或用户名。
2. 在下拉菜单中选择 **修改密码**。
3. 在弹出对话框中依次填写 **原密码**、**新密码**、**确认新密码**。
4. 点击 **确认** 按钮保存修改。
5. 修改成功后系统提示"密码修改成功，请重新登录"，并自动跳转至登录页面。

> 💡 **提示**：密码建议设置8位以上，包含大小写字母、数字及特殊符号，以提高安全性。

### 4.3 系统注销

**方法一（推荐）：**
1. 点击页面右上角的用户头像或用户名。
2. 在下拉菜单中点击 **退出登录**。
3. 系统弹出确认提示，点击 **确定**，即完成注销并跳转至登录页面。

**方法二：**
- 直接关闭浏览器标签页，下次访问系统时将需要重新登录（Token 过期后）。

> ⚠️ **注意**：在公共计算机上使用完毕后，务必点击退出登录，防止他人使用您的账号操作系统。

## 5. 仪表盘使用说明

【截图：仪表盘首页截图】

仪表盘是系统首页，提供企业资产数据的全局概览，帮助管理人员快速掌握资产动态。

### 5.1 统计卡片区域

仪表盘顶部展示四个核心统计卡片：

| 卡片名称     | 说明                                 |
| ------------ | ------------------------------------ |
| 资产总数     | 系统中所有在管资产的总数量           |
| 今日新增     | 当天新入库或新登记的资产数量         |
| 部门数量     | 系统中已配置的部门总数               |
| 维修中资产   | 当前状态为"维修中"的资产数量         |

> 💡 **提示**：点击各统计卡片可快速跳转至对应的资产列表页，并自动筛选相应状态。

### 5.2 图表区域

【截图：仪表盘图表区域截图】

仪表盘包含以下四种图表，均基于 ECharts 实现，支持鼠标悬停查看详细数据：

#### 5.2.1 资产状态分布饼图

- 以饼图形式展示各状态（采购中、闲置、使用中、维修中、报废）的资产数量和占比。
- 鼠标悬停在扇形区域可查看具体数值。
- 点击图例可隐藏/显示对应状态的数据。

#### 5.2.2 部门资产分布柱状图

- 横轴为各部门名称，纵轴为资产数量。
- 直观对比各部门持有资产的数量差异。

#### 5.2.3 资产趋势折线图

- 展示近期（如近12个月）资产数量的变化趋势。
- 可用于分析资产增长或减少的周期性规律。

#### 5.2.4 资产分类分布图

- 展示不同资产分类下的资产数量分布。
- 帮助管理者了解各类型资产的规模比例。

### 5.3 仪表盘数据刷新

- 仪表盘数据在每次进入页面时自动加载最新数据。
- 如需手动刷新，点击页面右上角的 **刷新** 按钮（或按 F5 刷新浏览器）即可重新加载数据。

## 6. 系统管理模块操作说明

系统管理模块入口位于左侧导航菜单的 **系统管理** 分组下，包含用户管理、角色管理、部门管理、权限管理四个子模块。

> ⚠️ **注意**：系统管理模块的所有操作仅限具有"系统管理员"角色或已被授予对应权限的用户执行。

### 6.1 用户管理

【截图：用户管理列表页面截图】

#### 6.1.1 查询用户

1. 点击左侧菜单 **系统管理 → 用户管理**，进入用户列表页面。
2. 在页面顶部搜索区域，可按以下条件进行筛选：
   - **用户名**：输入关键字进行模糊搜索。
   - **姓名**：输入真实姓名关键字搜索。
   - **状态**：选择"启用"或"禁用"过滤。
   - **所属部门**：从下拉框选择部门过滤。
3. 点击 **查询** 按钮，列表刷新显示符合条件的用户。
4. 点击 **重置** 按钮，清空所有筛选条件，恢复显示全部用户。

#### 6.1.2 新增用户

【截图：新增用户对话框截图】

1. 在用户管理列表页，点击页面右上方 **新增** 按钮，弹出新增用户对话框。
2. 填写以下必填信息：
   - **用户名**：设置登录账号，只能包含字母、数字和下划线。
   - **姓名**：用户真实姓名。
   - **密码**：设置初始密码。
   - **所属部门**：从下拉框选择用户所在部门。
3. 可选填写：手机号、邮箱、备注等信息。
4. 点击 **确定** 按钮保存，系统提示"新增成功"，用户列表刷新显示新用户。

> 💡 **提示**：新增用户后，请在"绑定角色"功能中为该用户分配相应角色，否则用户登录后将无法访问任何功能。

#### 6.1.3 修改用户信息

1. 在用户列表中，找到需要修改的用户，点击该行 **操作** 列的 **编辑** 按钮。
2. 弹出编辑对话框，已填入当前用户信息。
3. 修改需要变更的字段（用户名通常不可修改）。
4. 点击 **确定** 按钮保存修改，系统提示"修改成功"。

#### 6.1.4 删除用户

1. 在用户列表中，找到需要删除的用户，点击该行 **操作** 列的 **删除** 按钮。
2. 系统弹出确认提示"确定要删除该用户吗？此操作不可恢复"。
3. 点击 **确定**，系统删除该用户并从列表中移除。

> ⚠️ **注意**：删除用户操作不可恢复，请谨慎操作。建议对不再使用的账号执行"禁用"而非"删除"操作。

#### 6.1.5 启用 / 禁用用户

1. 在用户列表中，找到目标用户，点击 **操作** 列中的 **启用** 或 **禁用** 按钮。
2. 系统弹出确认提示，点击 **确定** 即可切换用户状态。
3. 禁用状态的用户将无法登录系统，但其历史记录仍然保留。

#### 6.1.6 重置用户密码

1. 在用户列表中，找到目标用户，点击 **操作** 列中的 **重置密码** 按钮。
2. 系统弹出确认提示，点击 **确定**。
3. 系统将密码重置为默认值（通常为 `123456`），并提示管理员告知用户及时修改密码。

#### 6.1.7 为用户分配角色

1. 在用户列表中，找到目标用户，点击 **操作** 列中的 **分配角色** 按钮。
2. 弹出角色分配对话框，左侧显示可选角色列表，右侧显示已分配角色。
3. 勾选需要分配的角色，点击 **>** 按钮移入已分配列表；或勾选已分配角色后点击 **<** 按钮移除。
4. 点击 **确定** 保存，用户权限立即生效（下次登录时生效）。

### 6.2 角色管理

【截图：角色管理页面截图】

#### 6.2.1 查询角色

1. 点击左侧菜单 **系统管理 → 角色管理**，进入角色列表页面。
2. 可在搜索框中输入角色名称关键字，点击 **查询** 进行筛选。

#### 6.2.2 新增角色

1. 点击 **新增** 按钮，弹出新增角色对话框。
2. 填写 **角色名称**（必填，如：资产管理员）和 **角色描述**（可选）。
3. 在权限树中勾选该角色可访问的菜单项和可操作的功能按钮。
4. 点击 **确定** 保存。

> 💡 **提示**：权限树中，勾选父节点会自动勾选所有子节点；取消父节点勾选会同时取消所有子节点。

#### 6.2.3 修改角色

1. 在角色列表中，点击目标角色行的 **编辑** 按钮。
2. 修改角色名称、描述或权限配置。
3. 点击 **确定** 保存，权限变更对所有持有该角色的用户立即生效。

#### 6.2.4 删除角色

1. 在角色列表中，点击目标角色行的 **删除** 按钮。

### 7.3 资产流转操作

资产流转操作用于改变资产的状态或归属，所有流转操作均会自动记录在流转历史中。

#### 7.3.1 入库

**适用场景**：将已采购但尚未登记的资产正式入库。

1. 在资产列表中，选中状态为"采购中"的资产，点击 **操作** 列的 **入库** 按钮。
2. 在弹出的入库对话框中，确认或填写：
   - **入库日期**（必填）
   - **入库仓库/存放位置**（必填）
   - **经办人**（必填）
   - **备注**（可选）
3. 点击 **确认入库**，资产状态变更为"闲置"。

#### 7.3.2 分配 / 调拨

**适用场景**：将闲置资产分配给具体部门或员工使用；或将资产从一个部门调拨至另一个部门。

1. 在资产列表中，选中状态为"闲置"的资产，点击 **分配/调拨** 按钮。
2. 填写以下信息：
   - **接收部门**（必填）
   - **接收人**（必填）
   - **调拨日期**（必填）
   - **操作类型**：分配或调拨
   - **备注**（可选）
3. 点击 **确认**，资产状态变更为"使用中"，归属部门和责任人自动更新。

#### 7.3.3 归还

**适用场景**：员工离职或部门不再使用某资产时，将资产归还入库。

1. 在资产列表中，选中状态为"使用中"的资产，点击 **归还** 按钮。
2. 填写归还日期和备注。
3. 点击 **确认归还**，资产状态变更为"闲置"，部门和责任人清空。

#### 7.3.4 报废

**适用场景**：资产超出使用年限或损毁无法修复，需要办理报废处置。

1. 在资产列表中，选中目标资产，点击 **报废** 按钮。
2. 填写：
   - **报废日期**（必填）
   - **报废原因**（必填）
   - **经办人**（必填）
3. 点击 **确认报废**，资产状态变更为"报废"，不再计入有效资产统计。

> ⚠️ **注意**：报废操作不可逆转，请在操作前确认资产确实符合报废条件。

#### 7.3.5 送修

**适用场景**：资产出现故障，需送往维修部门或外部维修商进行维修。

1. 在资产列表中，选中目标资产，点击 **送修** 按钮。
2. 填写：
   - **故障描述**（必填）
   - **送修日期**（必填）
   - **维修负责人**（必填）
3. 点击 **确认送修**，资产状态变更为"维修中"。
4. 维修完成后，在报修管理模块中完成维修记录，资产状态恢复为"闲置"或"使用中"。

## 9. 盘点管理操作说明

【截图：盘点管理页面截图】

### 9.1 新建盘点计划

1. 点击左侧菜单 **盘点管理**，进入盘点管理页面，点击 **新建盘点** 按钮。
2. 在弹出对话框中配置盘点计划：

   | 字段         | 说明                                         |
   | ------------ | -------------------------------------------- |
   | 盘点名称     | 如"2025年第一季度资产盘点"                   |
   | 盘点类型     | 全面盘点：对所有资产盘点；抽样盘点：随机抽取部分资产；专项盘点：针对特定分类或部门 |
   | 盘点范围     | 按部门或资产分类选择本次盘点涵盖的资产范围   |
   | 计划开始时间 | 从日期选择器选择                             |
   | 计划结束时间 | 从日期选择器选择                             |
   | 盘点负责人   | 指定本次盘点的负责人                         |
   | 备注         | 可填写盘点说明                               |

3. 点击 **确定** 保存，系统自动根据盘点范围生成待盘点的资产清单，盘点计划状态为"待开始"。

### 9.2 执行盘点

【截图：盘点执行页面截图】

1. 在盘点计划列表中，找到状态为"待开始"的计划，点击 **开始盘点** 按钮，计划状态变为"进行中"。
2. 点击 **执行盘点** 进入盘点操作页面，页面展示本次需要盘点的资产清单。
3. 对清单中每一项资产逐一进行核查：
   - **正常**：资产实物与系统记录一致，点击 **正常** 按钮确认。
   - **异常**：资产实物与记录不符（如丢失、损坏），点击 **异常** 按钮，并在弹出框中填写异常说明。
   - **未找到**：实物不在对应位置，标记为"未找到"。
4. 完成所有资产核查后，点击 **提交盘点结果** 按钮，系统弹出确认提示。
5. 点击 **确认提交**，盘点计划状态变更为"已完成"，系统自动生成盘点报告。

> 💡 **提示**：支持多人同时参与盘点，各人分别负责不同资产的核查，最终汇总结果。

### 9.3 查看盘点结果

【截图：盘点结果报告截图】

1. 在盘点计划列表中，点击已完成的盘点计划行，或点击 **查看结果** 按钮。
2. 进入盘点结果页面，展示以下统计信息：

   | 统计项     | 说明                           |
   | ---------- | ------------------------------ |
   | 应盘数量   | 本次计划盘点的资产总数         |
   | 实盘数量   | 已完成核查的资产数量           |
   | 正常数量   | 状态正常的资产数量             |
   | 异常数量   | 标记为异常的资产数量           |
   | 未找到数量 | 实物未找到的资产数量           |
   | 盘点完成率 | 实盘数量 / 应盘数量 × 100%     |

3. 点击 **导出报告** 可将盘点结果导出为 Excel 文件，用于存档和上报。
4. 在异常资产列表中，点击每条异常记录可查看详情并后续跟进处理。

## 10. 报修管理操作说明

【截图：报修管理页面截图】

报修管理模块支持资产故障申报、审批、维修记录全流程管理。

### 10.1 发起报修申请

1. 点击左侧菜单 **报修管理**，进入报修管理页面，点击 **新增报修** 按钮。
2. 在弹出表单中填写以下信息：

   | 字段         | 是否必填 | 说明                                    |
   | ------------ | -------- | --------------------------------------- |
   | 报修资产     | 必填     | 从资产列表中搜索并选择故障资产          |
   | 故障描述     | 必填     | 详细描述故障现象，便于维修人员快速定位  |
   | 报修类型     | 必填     | 硬件故障/软件故障/其他                  |
   | 优先级       | 必填     | 紧急/高/中/低                           |
   | 报修人       | 自动填充 | 当前登录用户                            |
   | 期望完成时间 | 选填     | 希望维修完成的目标日期                  |
   | 附件         | 选填     | 可上传故障照片或相关文件                |

3. 填写完毕后，点击 **提交** 按钮，报修单状态为"待审批"，系统自动通知审批人。

### 10.2 审批报修申请

**操作角色**：具有报修审批权限的管理人员

1. 在报修管理列表中，点击状态为"待审批"的报修单，或通过系统通知进入审批页面。
2. 查看报修申请详情，包括资产信息、故障描述、报修优先级等。
3. 执行审批操作：
   - **同意**：点击 **通过** 按钮，可在审批意见框中填写备注，点击 **确认通过**，报修单状态变为"待维修"。
   - **拒绝**：点击 **拒绝** 按钮，必须在拒绝原因框中填写拒绝理由，点击 **确认拒绝**，报修单状态变为"已拒绝"，申请人收到通知。

### 10.3 维修处理

**操作角色**：维修人员

1. 在报修管理列表中，筛选状态为"待维修"的报修单，找到负责维修的工单。
2. 点击 **开始维修** 按钮，报修单状态变为"维修中"，对应资产状态自动更新为"维修中"。
3. 维修完成后，点击 **完成维修** 按钮，填写以下维修记录：

   | 字段       | 是否必填 | 说明                       |
   | ---------- | -------- | -------------------------- |
   | 维修人     | 必填     | 实际执行维修的人员         |
   | 维修完成日期 | 必填   | 实际完成维修的日期         |
   | 维修费用   | 选填     | 维修产生的费用（元）       |
   | 维修结果   | 必填     | 维修成功/更换配件/无法修复 |
   | 维修详情   | 必填     | 描述维修过程和处理方案     |

4. 点击 **确认完成**，报修单状态变为"已完成"，资产状态根据维修结果自动更新：
   - 维修成功 → 资产恢复为"闲置"或"使用中"
   - 无法修复 → 提示是否对该资产执行报废操作

### 10.4 查看报修历史

1. 在报修管理列表中，可按报修单状态、资产名称、报修人、时间范围等条件筛选查询。
2. 点击任意报修单行可查看完整的处理流程记录，包括申请、审批、维修各阶段的时间节点和操作人。

## 11. 采购管理操作说明

【截图：采购管理页面截图】

采购管理模块实现从采购申请到资产入库的完整采购流程管理。

### 11.1 新建采购申请

1. 点击左侧菜单 **采购管理**，进入采购管理页面，点击 **新建采购单** 按钮。
2. 填写采购单基本信息：

   | 字段       | 是否必填 | 说明                             |
   | ---------- | -------- | -------------------------------- |
   | 采购单编号 | 自动生成 | 系统自动生成，格式如 PO202501001 |
   | 供应商     | 必填     | 填写或从供应商库选择             |
   | 采购日期   | 必填     | 计划采购的日期                   |
   | 申请部门   | 必填     | 自动填入申请人所在部门           |
   | 申请人     | 必填     | 自动填入当前登录用户             |
   | 备注       | 选填     | 采购背景或特殊说明               |

3. 在 **采购明细** 区域，点击 **添加明细行** 按钮，逐条填写采购资产信息：

   | 字段       | 是否必填 | 说明                                 |
   | ---------- | -------- | ------------------------------------ |
   | 资产名称   | 必填     | 拟采购的资产名称                     |
   | 资产分类   | 必填     | 从分类树中选择                       |
   | 规格型号   | 必填     | 型号描述                             |
   | 单价（元） | 必填     | 单件资产价格                         |
   | 数量       | 必填     | 采购数量，系统自动计算小计金额       |
   | 备注       | 选填     | 该明细行的特殊说明                   |

4. 可点击 **添加明细行** 继续添加多种资产，合计金额自动汇总显示在底部。
5. 填写完毕后，点击 **提交申请** 按钮，采购单状态变为"待审批"，自动通知审批人。

> 💡 **提示**：提交前可点击 **保存草稿** 按钮将未完成的采购单保存为草稿，下次继续填写。

### 11.2 采购审批

**操作角色**：具有采购审批权限的管理人员

1. 在采购管理列表中，找到状态为"待审批"的采购单，点击 **查看详情** 按钮。
2. 审查采购单内容，核实采购必要性、数量、金额等。
3. 执行审批操作：
   - **通过**：点击 **审批通过** 按钮，填写审批意见（可选），点击 **确认**，采购单状态变为"已审批"。
   - **退回**：点击 **退回修改** 按钮，填写退回原因（必填），申请人可修改后重新提交。
   - **拒绝**：点击 **拒绝** 按钮，填写拒绝原因（必填），采购单状态变为"已拒绝"。

### 11.3 采购入库

**操作角色**：资产管理员

审批通过的采购单在实物到货后，需执行入库操作将资产正式登记入系统台账。

1. 在采购管理列表中，找到状态为"已审批"的采购单，点击 **入库处理** 按钮。
2. 进入入库处理页面，系统展示该采购单所有明细行。
3. 对每一条明细行逐一确认入库：
   - 确认实际到货数量（与采购数量核对）。
   - 填写资产存放位置。
   - 如有序列号、固定资产编码等信息，在对应字段填写。
4. 全部明细确认后，点击 **确认入库** 按钮。
5. 系统自动在资产台账中为每件资产创建记录，状态为"闲置"，同时采购单状态更新为"已入库"。

> 💡 **提示**：如到货数量与采购数量不符（部分到货），系统支持部分入库，剩余数量待后续补入库。

### 11.4 查看采购记录

1. 在采购管理列表中，可按采购单编号、供应商、状态、申请时间等条件筛选查询。
2. 点击任意采购单可查看完整的审批流程和入库详情。

## 12. Excel 导入导出操作说明

### 12.1 资产数据导出

【截图：资产导出操作截图】

**操作步骤：**

1. 进入 **资产管理 → 资产信息** 页面。
2. 如需导出筛选后的资产，先设置筛选条件并点击 **查询**；如需导出全部资产，保持默认状态。
3. 点击页面右上角 **导出** 按钮，弹出导出设置对话框。
4. 选择导出范围：
   - **当前页数据**：仅导出当前显示的页面数据。
   - **全部数据**：导出所有符合筛选条件的数据（数据量大时可能需要等待）。
5. 选择要导出的字段（可勾选/取消勾选字段）。
6. 点击 **确认导出**，浏览器自动下载 Excel 文件（文件名包含导出日期）。

### 12.2 资产数据批量导入

【截图：资产导入操作截图】

**前提条件**：使用系统提供的标准模板文件进行导入。

**步骤一：下载导入模板**

1. 在资产信息页面，点击 **导入** 按钮旁的下拉箭头，选择 **下载模板**。
2. 浏览器下载名为 `资产导入模板.xlsx` 的文件，保存到本地。

**步骤二：填写导入数据**

1. 打开下载的 Excel 模板文件。
2. 按照模板第一行的字段说明，逐行填写待导入的资产信息。
3. 注意事项：
   - **资产分类** 列须填写系统中已存在的分类名称（完整路径，如：IT设备/计算机/笔记本电脑）。
   - **使用部门** 列须填写系统中已存在的部门名称。
   - **资产状态** 列须填写系统支持的状态值：闲置、使用中、维修中、采购中、报废。
   - 不可修改模板中标注"*"的必填列的列名。
4. 填写完毕后保存 Excel 文件。

**步骤三：上传并导入**

1. 在资产信息页面，点击 **导入** 按钮，弹出导入对话框。
2. 点击 **选择文件** 按钮，从本地选择已填好的 Excel 文件。
3. 点击 **开始导入** 按钮，系统进行数据校验。
4. 校验通过后，系统显示"即将导入 XX 条记录"的确认提示。
5. 点击 **确认导入**，系统开始处理导入，完成后显示导入结果报告（成功数/失败数/失败原因）。

> ⚠️ **注意**：单次导入建议不超过 5000 条记录，超出数量建议分批次导入。
>
> ⚠️ **注意**：导入过程中请勿关闭浏览器或刷新页面，以免导入中断。

**常见导入失败原因及处理：**

| 错误提示             | 原因                         | 解决方法                     |
| -------------------- | ---------------------------- | ---------------------------- |
| 资产名称不能为空     | 资产名称字段未填写           | 检查并补充资产名称           |
| 资产分类不存在       | 填写的分类名称在系统中不存在 | 先在分类管理中创建对应分类   |
| 部门不存在           | 填写的部门名称不正确         | 确认并更正部门名称           |
| 资产状态值不合法     | 状态值填写错误               | 参照模板填写合法状态值       |
| 采购金额格式错误     | 金额字段包含非数字字符       | 仅填写纯数字，不含货币符号   |

## 13. 常见问题与解决方案

### Q1：登录时提示"用户名或密码错误"

**原因分析**：
- 用户名或密码输入有误（注意大小写区分）。
- 账号已被禁用。
- 系统 JWT 密钥配置问题。

**解决方法**：
1. 确认输入的用户名和密码是否正确，特别注意大写锁定（Caps Lock）是否开启。
2. 联系系统管理员确认账号是否处于启用状态。
3. 若多次尝试仍失败，联系管理员重置密码。

---

### Q2：登录成功后页面空白或菜单不显示

**原因分析**：
- 当前用户未被分配任何角色，无可访问的菜单权限。
- 浏览器缓存问题。

**解决方法**：
1. 联系系统管理员，确认账号已被分配了具有菜单权限的角色。
2. 尝试按 `Ctrl+Shift+Delete` 清除浏览器缓存后重新登录。
3. 尝试切换浏览器（如使用 Chrome）访问。

---

### Q3：无法对资产进行流转操作，按钮显示为灰色

**原因分析**：
- 当前用户角色不具有该操作的权限。
- 资产当前状态不允许执行该操作（如已报废的资产无法再进行分配）。

**解决方法**：
1. 检查资产当前状态是否符合操作前提（参考各流转操作的"适用场景"说明）。
2. 如状态正确，联系系统管理员确认账号角色是否具有相应操作权限。

---

### Q4：Excel 导入时所有数据均导入失败

**原因分析**：
- 未使用系统提供的标准模板文件。
- 模板表头被修改。
- 文件格式不正确（仅支持 `.xlsx` 格式）。

**解决方法**：
1. 重新下载最新版模板文件，不要修改表头行。
2. 确认文件保存为 `.xlsx` 格式（不支持 `.xls` 或 `.csv`）。
3. 检查数据行是否从第2行开始填写（第1行为表头）。

---

### Q5：报表数据与实际情况不符

**原因分析**：
- 数据缓存未刷新。
- 资产状态变更操作未完成（如报修单未正式关闭，资产状态未更新）。

**解决方法**：
1. 刷新页面后重新查看数据。
2. 检查相关资产的操作流程是否完整（如报修是否已点击"完成维修"）。
3. 如问题持续，联系管理员检查数据一致性。

---

### Q6：系统访问速度缓慢

**原因分析**：
- 服务器负载过高。
- 网络连接不稳定。
- 浏览器缓存数据过多。

**解决方法**：
1. 检查本地网络连接是否正常。
2. 清除浏览器缓存后重试。
3. 避开系统使用高峰时段访问（如每月月底盘点期间）。
4. 联系系统管理员检查服务器状态。

---

### Q7：忘记登录密码

**解决方法**：
1. 联系系统管理员，由管理员在用户管理页面执行 **重置密码** 操作。
2. 密码重置后使用默认密码（`123456`）登录，登录后立即修改密码。

## 14. 系统维护说明

### 14.1 日常运维建议

#### 14.1.1 数据备份

- **备份频率**：建议每日自动备份数据库，保留最近30天的备份。
- **备份命令**：
  ```bash
  mysqldump -u eams_user -p eams > eams_backup_$(date +%Y%m%d).sql
  ```
- **备份存储**：备份文件建议存储在独立于数据库服务器的存储介质或云存储上。

#### 14.1.2 日志管理

- 系统运行日志默认存储在 `logs/` 目录下。
- 建议定期（每月）清理或归档30天前的日志文件，防止磁盘空间不足。
- 遇到系统异常时，优先查看 `logs/error.log` 文件进行问题定位。

#### 14.1.3 系统监控

- 建议监控以下关键指标：
  - JVM 堆内存使用率（建议不超过 80%）
  - MySQL 数据库连接数及慢查询日志
  - 磁盘空间使用率（建议不超过 85%）

### 14.2 系统升级

1. 升级前，务必进行完整的数据库备份。
2. 查看升级说明文档，确认是否有数据库 Schema 变更脚本需要执行。
3. 停止后端服务：
   ```bash
   # 查找进程PID
   ps -ef | grep eams
   # 停止进程
   kill <PID>
   ```
4. 替换新版本的 JAR 文件，并执行数据库升级脚本（如有）。
5. 前端升级：将新版本 `dist/` 目录文件替换到 Web 服务器对应目录。
6. 重新启动服务，验证系统功能正常。

### 14.3 用户权限定期审查

- 建议每季度组织一次用户权限审查，包括：
  - 清理已离职员工账号（禁用或删除）。
  - 核查各用户的角色分配是否与当前岗位职责匹配。
  - 检查是否有不必要的高权限账号。

### 14.4 数据一致性检查

- 定期（建议每月）通过数据库查询工具核查以下数据一致性：
  - 资产状态与流转记录是否对应。
  - 采购入库数量与资产台账数量是否一致。
  - 报修单关闭状态与对应资产状态是否匹配。

### 14.5 技术支持

如遇到系统操作问题或系统故障，请通过以下渠道获取支持：

- **系统管理员**：联系本企业系统管理员解决权限、账号等问题。
- **技术支持邮件**：发送问题描述及截图至技术支持团队邮箱。
- **问题反馈**：在系统右上角点击 **帮助与反馈**，提交问题报告。

> 💡 **提示**：提交技术支持请求时，请附上问题描述、操作步骤、错误截图及浏览器版本信息，有助于技术团队快速定位和解决问题。
