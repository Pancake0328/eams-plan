# 盘点管理更新说明（2026-02-07）

## 上次修改内容
- 移除前端侧边栏“系统管理”及其下级菜单入口。
- 盘点计划创建与执行盘点默认使用当前登录用户作为创建人/盘点人，输入框只读。
- 盘点详情改为分页查询明细，新增接口：`GET /inventory/{inventoryId}/details`。
- 专项盘点新增按资产分类筛选：创建时选择分类，仅生成该分类及其子分类的资产明细。
- 全面盘点与专项盘点创建明细新增批量插入与部门名称缓存，提升创建性能。

## 本次修复与补充
- 抽样盘点新增“抽样数量（sampleCount）”，按填写数量随机抽取资产。
- 修复全量盘点 `ExecutorType` 事务冲突：批量插入改为批量 SQL 提交，避免事务内切换执行器。

## 相关数据字段
- `asset_inventory` 新增：
  - `sample_count` 抽样数量（抽样盘点使用）
  - `category_id` 专项盘点分类 ID

## 前端调整说明
- 创建盘点计划：
  - 抽样盘点显示“抽样数量”输入框并必填。
  - 专项盘点显示“盘点分类”选择器并必填。
- 盘点详情：
  - 抽样盘点展示抽样数量。
  - 明细分页加载，减少一次性渲染与传输量。
