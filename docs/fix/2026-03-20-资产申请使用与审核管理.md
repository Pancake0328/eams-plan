# 2026-03-20 资产申请使用与审核管理

## 变更目标

在“全部资产”的操作菜单中新增“申请使用”，允许普通用户主动发起资产使用申请；新增“申请审核管理”页面用于审核通过/拒绝，形成完整申请流。

---

## 功能变更

## 1）全部资产新增“申请使用”

涉及：`frontend/src/views/AssetManagement.vue`

- 在“更多操作”中新增 `申请使用`（权限：`asset:usage:apply`）；
- 仅当资产状态为“闲置（1）”时可点击；
- 弹窗新增“申请原因”必填项；
- 提交后调用 `POST /asset-usage-applications` 创建申请。

## 2）新增申请审核管理页面

涉及：

- `frontend/src/views/UsageApplicationManagement.vue`
- `frontend/src/router/index.ts`
- `frontend/src/layout/MainLayout.vue`

新增路由：

- `/usage-applications`（权限：`asset:usage:list`）

页面能力：

- 申请列表分页查询（按状态/资产编号/资产名称/申请人筛选）；
- 对待审核记录执行通过/拒绝（权限：`asset:usage:audit`）。

## 3）新增后端申请模块

涉及：

- `src/main/java/com/eams/asset/entity/AssetUsageApplication.java`
- `src/main/java/com/eams/asset/mapper/AssetUsageApplicationMapper.java`
- `src/main/java/com/eams/asset/dto/UsageApplicationCreateRequest.java`
- `src/main/java/com/eams/asset/dto/UsageApplicationAuditRequest.java`
- `src/main/java/com/eams/asset/dto/UsageApplicationPageQuery.java`
- `src/main/java/com/eams/asset/vo/UsageApplicationVO.java`
- `src/main/java/com/eams/asset/service/AssetUsageApplicationService.java`
- `src/main/java/com/eams/asset/service/impl/AssetUsageApplicationServiceImpl.java`
- `src/main/java/com/eams/asset/controller/AssetUsageApplicationController.java`

新增接口：

- `POST /asset-usage-applications`：创建申请
- `GET /asset-usage-applications`：分页查询申请
- `PUT /asset-usage-applications/{id}/audit`：审核申请

关键业务规则：

- 创建申请前校验资产存在且为闲置状态；
- 申请人必须存在且绑定部门；
- 同一资产不可重复提交“待审核”申请；
- 审核通过时二次校验资产仍为闲置，再复用 `assetRecordService.allocateAsset` 自动分配给申请人；
- 审核拒绝仅变更申请状态，不改资产状态。

## 白屏问题修复（申请审核管理）

问题现象：进入“申请审核管理”出现白屏。

排查结论：

- `UsageApplicationManagement.vue` 组件可正常打包；
- 路由页面存在，但历史环境可能访问了旧路径 `/usage-application`（单数），与当前正式路径 `/usage-applications`（复数）不一致；
- 原路由未配置兜底，未匹配路径时 `router-view` 无内容，表现为白屏。

修复内容：

- 在 `frontend/src/router/index.ts` 新增兼容别名路由：`/usage-application -> /usage-applications`；
- 增加兜底路由：`/:pathMatch(.*)* -> /no-access`，避免未知路径白屏；
- 在 `frontend/src/layout/MainLayout.vue` 补充 `/usage-application` 面包屑映射，保证跳转展示一致。

回归验证：

- 前端执行 `npm exec vite build` 通过；
- 访问 `/usage-applications` 与 `/usage-application` 均可正常进入“申请审核管理”。

## 员工资产自助端（新增）

### 目标

将普通用户从“后台管理风格”中分离，新增独立的员工资产自助入口，提升自助办理体验与视觉友好度。

### 前端改动

新增独立布局与页面：

- `frontend/src/layout/PortalLayout.vue`
- `frontend/src/views/portal/PortalHome.vue`
- `frontend/src/views/portal/PortalAssetCenter.vue`
- `frontend/src/views/portal/PortalMyAssets.vue`
- `frontend/src/views/portal/PortalMyApplications.vue`
- `frontend/src/views/portal/PortalMyRepairs.vue`

路由与守卫增强：

- `frontend/src/router/index.ts` 新增 `/portal/**` 路由域；
- 新增 `portalOnly` 元数据；
- 登录后根据权限自动分流：
  - 纯员工权限用户默认进入 `/portal/home`；
  - 管理用户仍进入后台管理路由；
- 纯员工权限用户访问后台路由时自动重定向至自助端。

### 后端接口增强

新增自助端查询接口：

- `GET /assets/portal`（权限：`asset:portal:view`）  
  员工可浏览公司资产（默认仅展示状态 1/2/3，支持按编号/名称/分类/状态筛选）。
- `GET /asset-usage-applications/my`（权限：`asset:usage:my:list`）  
  员工仅查看本人申请记录。

对应代码：

- `AssetInfoController` / `AssetInfoService` / `AssetInfoServiceImpl`
- `AssetUsageApplicationController` / `AssetUsageApplicationService` / `AssetUsageApplicationServiceImpl`

### 权限与菜单脚本

更新 `sql/测试脚本/sys_menu.sql`、`sql/测试脚本/sys_role_menu.sql`，新增：

- `asset:portal:view`（员工自助首页/公司资产菜单）
- `asset:usage:my:list`（我的申请菜单）
- 自助端目录：`/portal`
- 自助端菜单：`/portal/home`、`/portal/assets`、`/portal/my-assets`、`/portal/my-applications`、`/portal/my-repairs`
- 自助端按钮：公司资产页“申请使用”

并将上述权限关联到现有普通角色（及管理角色），保证开箱可访问。

### 验证结果

- 前端：`npm exec vite build` 通过；
- 后端：`mvn test -q` 通过。
