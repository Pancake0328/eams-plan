# 2026-03-20 RBAC 菜单与权限修复

## 背景

本次根据使用反馈，对系统菜单结构与 RBAC 做集中修复，重点解决：

1. 登录后未稳定落到欢迎页；
2. 菜单管理中新建目录后，左侧菜单不展示；
3. 菜单结构未对齐目标分组；
4. RBAC 细节存在完成度不足与一致性问题。

---

## 一、问题说明（先说明）

### 1. 登录落地页不稳定

- 现状：登录后统一 `router.push('/')`，但 `/` 没有稳定重定向，实际落点依赖守卫分支。
- 风险：不同入口（刷新、直达、登录跳转）行为不一致，造成“有时不是欢迎页”的体验问题。

### 2. 新建目录后左侧不展示（核心问题）

- 现状：菜单管理创建新目录仅写入 `sys_menu`，未写 `sys_role_menu`。
- 风险：菜单树按“角色-菜单关联”构建，新目录对任何角色都不可见，导致左侧不显示。

### 3. 菜单结构与目标不完全一致

- 现状：报修管理为顶层 `MENU`，无法自然承载“全部报修/我的报修”双菜单；盘点与生命周期混在同一目录层级。
- 风险：菜单信息架构与业务认知不一致，授权与导航都不直观。

### 4. RBAC 完成度不足（不合理点）

- 用户权限加载未过滤禁用菜单，可能发放已禁用权限；
- 角色删除未清理 `sys_user_role`/`sys_role_menu`，会留下脏关联；
- 用户菜单树未严格按启用 + 可见过滤，且祖先链路处理不完整。

---

## 二、修复内容（后修复）

## 1) 登录与欢迎页

涉及文件：`frontend/src/router/index.ts`

- 为根路由增加 `redirect: '/welcome'`；
- 守卫中对 `to.path === '/'` 统一使用 `resolveFirstAccessiblePath(permissionStore)`；
- 首页可访问顺序重排，优先欢迎页，随后按新菜单结构排序。

结果：登录后进入 `/` 将稳定到欢迎页（或第一个有权限页面）。

## 2) 新建目录可见性修复（根因修复）

涉及文件：`src/main/java/com/eams/system/service/impl/MenuServiceImpl.java`

- `createMenu` 后新增自动角色继承：
  - 若有父菜单：继承父菜单已有角色关联；
  - 若为顶级菜单：继承拥有 `system:permission:add` 的角色（即可管理菜单的角色）。
- 批量写入 `sys_role_menu` 后，逐角色清理权限缓存；
- `deleteMenu` 时同步删除角色菜单关联并清理受影响缓存，避免残留授权。

结果：在菜单管理新建目录/菜单后，拥有父级权限的角色可立即在左侧看到。

## 3) 菜单结构调整（目标结构落地）

涉及脚本：`sql/开发中脚本/menu_rbac_structure_20260320.sql`

完成以下调整：

- 顶层结构调整为：
  1. 仪表盘
  2. 系统管理（部门、角色、菜单、用户）
  3. 资产管理（全部资产、持有资产、资产分类、流转记录）
  4. 采购管理
  5. 报修管理（全部报修、我的报修）
  6. 盘点管理
  7. 资产生命周期
- 将“报修管理(id=18)”从 `MENU` 调整为 `DIR`，新增“全部报修(id=97)”承载 `repair:list`；
- 保留并修正“我的报修(id=96)”；
- 保留并修正“持有资产(id=95)”；
- 普通用户从全量资产/全量报修切到“持有资产 + 我的报修”；
- 管理角色补齐新增菜单授权（`ADMIN`/`ADMIN-SYS`/`ASSET_MANAGER`）。

## 4) RBAC 一致性补强

涉及文件：

- `src/main/java/com/eams/system/service/impl/PermissionServiceImpl.java`
- `src/main/java/com/eams/system/service/impl/RoleServiceImpl.java`

修复点：

- `getUserPermissions` 仅返回启用菜单权限；
- `getUserRoleIds` 仅返回启用角色；
- `getUserMenuTree` 使用“授权菜单向上回溯父链”构建可展示树，并过滤 `status=1 && visible=1`；
- `buildMenuTree` 使用 `Objects.equals`，避免空值比较风险；
- 删除角色时清理 `sys_role_menu` + `sys_user_role` 并刷新关联用户缓存；
- 角色权限读取时过滤禁用菜单。
