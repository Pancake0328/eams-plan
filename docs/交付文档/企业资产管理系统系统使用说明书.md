## 2. 系统访问与登录

### 2.1 访问地址
- 后端 API 默认：`http://<服务器>:9998/api`
- Swagger（接口文档）：`http://<服务器>:9998/api/swagger-ui.html`
- 前端管理端：由 Nginx 或 Vite Dev Server 提供（以部署为准）

### 2.2 浏览器要求
推荐使用：Chrome、Edge（最新版或近 2 个大版本）。

### 2.3 登录
1. 打开系统登录页。
2. 输入账号与密码。
3. 点击“登录”。

【截图：登录页面】

#### 2.3.1 登录成功后的初始化
登录成功后系统会自动加载：
- 用户基础信息（头像/昵称等）
- 当前用户权限集合（permission codes）
- 当前用户菜单树（用于左侧导航渲染）

#### 2.3.2 登录失败处理
- 密码错误：请确认大小写与输入法状态；
- 账号被禁用：联系系统管理员启用；
- token 过期：重新登录。

### 2.4 退出登录
1. 点击右上角用户下拉菜单。
2. 选择“退出登录”。
3. 在确认框中点击“确定”。

【截图：退出登录确认框】

## 3. 权限与菜单说明

### 3.1 菜单可见性
左侧菜单根据用户所拥有的“菜单权限”动态展示：
- 未授权的菜单不会显示；
- 直接输入 URL 访问无权限页面时，系统将自动跳转到“首个可访问页面”或“无权限访问”页面。

### 3.2 按钮级权限
页面内的关键操作按钮（新增/编辑/删除/审批/开始/完成等）通过按钮权限控制：
- 没有对应权限时，按钮不会显示或不可用；
- 即使按钮隐藏，后端接口也会做强制鉴权（最终以接口鉴权为准）。

### 3.3 典型权限示例
- 用户列表：`system:user:list`
- 新增用户：`system:user:add`
- 创建盘点：`inventory:create`
- 报修审批：`repair:approve`

> 完整权限清单：`docs\\开发中文档\\菜单权限清单.md`

## 5. 模块操作指南

> 说明：以下以 `frontend/src/router/index.ts` 中的页面为准。

### 5.1 仪表盘（仪表盘统计）
入口：左侧菜单“仪表盘”。

**主要功能**
- 查看资产总览指标
- 查看按部门/分类/状态的分布
- 查看时间趋势图

**操作步骤**
1. 进入“仪表盘”页面。
2. 观察顶部指标卡（如总资产数、在用数等）。
3. 查看各图表并根据提示切换维度（若支持）。

【截图：仪表盘页面】

**常见问题**
- 图表为空：检查是否有权限 `dashboard:view`；检查数据库是否存在资产数据。

### 5.2 用户管理
入口：默认首页或菜单“用户管理”。

**主要功能**
- 用户列表查询
- 新增/编辑用户
- 启用/禁用
- 重置密码
- 分配角色

**操作步骤**
1. 进入“用户管理”页面。
2. 使用筛选条件（用户名/部门/状态等）查询。
3. 点击“新增用户”填写表单并提交。
4. 在用户行操作区执行“编辑/禁用/重置密码/分配角色”。

【截图：用户管理-列表】
【截图：用户管理-新增/编辑弹窗】
【截图：用户管理-分配角色弹窗】

**注意事项**
- 分配角色后，用户权限随角色权限变化；
- 禁用用户后无法登录。

### 5.3 部门管理
入口：菜单“部门管理”。

**主要功能**
- 维护部门树
- 新增子部门/编辑部门/删除部门

**操作步骤**
1. 进入“部门管理”。
2. 在部门树上选择节点。
3. 点击“新增子部门/编辑/删除”。
4. 提交表单后刷新树结构。

【截图：部门管理-树形结构与操作按钮】

**注意事项**
- 删除部门前需确保无子部门且无用户绑定（具体以系统校验提示为准）。

### 5.4 角色管理
入口：菜单“角色管理”。

**主要功能**
- 创建/编辑/删除/启用/禁用角色
- 为角色分配菜单/按钮权限

**操作步骤**
1. 进入“角色管理”。
2. 点击“新增角色”填写角色编码与名称。
3. 在角色列表中点击“权限配置”，勾选菜单树与按钮权限。
4. 保存后权限即时生效（可能需要重新加载权限/重新登录）。

【截图：角色管理-列表】
【截图：角色管理-权限配置（菜单树勾选）】

**建议**
- 角色编码建议使用 `ROLE_xxx` 或 `asset_manager` 等清晰命名；
- 对关键操作（删除/报废/审批）建议单独权限点以便审计。

### 5.5 菜单管理（菜单/权限点）
入口：菜单“菜单管理”。

**主要功能**
- 维护菜单树（目录/菜单/按钮）
- 配置 `permissionCode`（权限标识）

**操作步骤**
1. 进入“菜单管理”。
2. 点击“新增根节点”或在节点上新增子节点。
3. 选择类型：目录（DIR）/菜单（MENU）/按钮（BUTTON）。
4. 填写名称、排序、可见性与权限标识（按钮/菜单一般都需配置）。

【截图：菜单管理-树/表格与新增弹窗】

**注意事项**
- 权限标识建议遵循 `模块:资源:动作`，例如：`system:user:add`。

### 5.6 资产分类管理
入口：菜单“资产分类管理”。

**主要功能**
- 维护资产分类树

**操作步骤**
1. 进入“资产分类管理”。
2. 新增根分类或子分类。
3. 编辑分类名称与排序。
4. 删除分类（若存在子分类或已有资产挂载，系统通常会阻止删除）。

【截图：资产分类管理-树形列表】

### 5.7 资产信息管理（资产台账）
入口：菜单“资产信息管理”。

**主要功能**
- 资产列表查询与筛选
- 查看资产详情
- 执行资产流转（入库/分配/调拨/归还/送修/维修完成/报废）

**操作步骤（典型）**
1. 进入“资产信息管理”。
2. 使用筛选条件（分类/部门/状态/编码等）查询。
3. 点击“查看”进入详情。
4. 根据权限显示的操作按钮执行流转动作。

【截图：资产信息管理-列表】
【截图：资产详情】

**流转操作说明（示例）**
- 入库：通常来自采购入库或手工入库，入库后生成资产编码；
- 分配：选择目标部门/人员，确认后资产进入“在用/已分配”；
- 调拨：选择目标部门，确认后归属变更并记录历史；
- 送修/维修完成：与报修流程保持一致；
- 报废：不可逆操作，需谨慎。

## 6. 常见问题（FAQ）

### 6.1 看不到某个菜单/按钮
- 可能原因：当前账号无对应权限。
- 处理方法：联系系统管理员在“角色管理/菜单管理”中分配权限并重新登录。

### 6.2 登录后提示 token 过期
- 可能原因：token 超时（当前配置默认 1800 秒）。
- 处理方法：重新登录；如需延长有效期请由运维修改后端配置。

### 6.3 操作提示 403 无权限
- 说明：后端接口鉴权未通过。
- 处理方法：检查角色权限是否包含对应 `permission_code`。

### 6.4 数据不更新/页面异常
- 刷新页面或重新登录；
- 检查网络与后端服务是否正常；
- 联系运维查看后端日志。
