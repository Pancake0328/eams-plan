# EAMS 企业资产管理系统 - 技术栈总结

## 项目概述

EAMS (Enterprise Asset Management System) 是一个基于前后端分离架构的企业级资产管理系统，提供资产全生命周期管理能力，包括资产信息管理、资产分类、资产盘点、资产维护、采购管理、统计报表等功能。

---

## 一、开发平台与架构

### 1.1 系统架构
- **架构模式**: 前后端分离架构（Front-end & Back-end Separation）
- **后端架构**: 三层架构 (Controller → Service → Mapper)
- **前端架构**: 组件化架构 + 状态管理
- **数据交互**: RESTful API
- **认证方式**: JWT (JSON Web Token) 无状态认证

### 1.2 设计模式
- **DTO/VO 分离模式**: 请求参数、响应数据、实体严格分离
- **AOP 切面编程**: 操作日志自动记录
- **过滤器链模式**: Spring Security 安全过滤器链
- **策略模式**: 多种认证方式扩展支持
- **统一返回结果封装**: Result<T> 统一响应格式

## 十一、开发规范

### 11.1 后端规范

**代码规范:**
- 三层架构: Controller → Service → Mapper
- DTO/VO 严格分离
- 所有方法、字段必须有中文注释
- 统一使用 Result<T> 返回
- RESTful API 设计

**命名规范:**
- 类名: 大驼峰 (PascalCase)
- 方法名/变量名: 小驼峰 (camelCase)
- 常量: 全大写下划线分隔 (UPPER_SNAKE_CASE)
- 包名: 全小写

**数据库规范:**
- 表名: 小写下划线分隔
- 字段名: 小写下划线分隔
- 必须字段: id, create_time, update_time, deleted

### 11.2 前端规范

**代码规范:**
- 使用 TypeScript 编写代码
- 遵循 Vue 3 Composition API 风格
- 组件使用 `<script setup>` 语法
- 样式使用 scoped 避免污染
- 使用 Element Plus 组件库

**命名规范:**
- 组件名: 大驼峰 (PascalCase)
- 文件名: 中划线分隔 (kebab-case)
- 变量名/函数名: 小驼峰 (camelCase)
- 常量: 全大写下划线分隔 (UPPER_SNAKE_CASE)

## 十二、总结

EAMS 企业资产管理系统采用了成熟、主流的技术栈，具有以下特点:

1. **现代化技术栈**: Java 17 + Spring Boot 3 + Vue 3，使用最新稳定版本
2. **前后端分离**: 清晰的架构边界，方便团队协作和独立部署
3. **安全可靠**: Spring Security + JWT 认证，BCrypt 密码加密
4. **高性能**: MyBatis Plus + Redis 缓存，HikariCP 连接池
5. **易维护**: 统一规范、完善文档、Swagger API
6. **可扩展**: 模块化设计，支持功能扩展
7. **跨平台**: 支持 Windows、Linux、macOS 开发和部署
8. **企业级**: 完善的权限管理、日志记录、异常处理

该系统适合作为企业资产管理的基础平台，可根据实际业务需求进行定制化开发。
