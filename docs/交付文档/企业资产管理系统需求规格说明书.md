## 1. 引言

### 1.1 编写目的
本说明书用于对“企业资产管理系统（EAMS）”的业务范围、功能需求、数据需求、外部接口需求及非功能性需求进行统一、明确、可验收的描述，为后续的设计、开发、测试、验收与运维提供依据。

### 1.2 项目背景
在多数组织中，资产从采购、入库、领用、调拨、维修、盘点到报废的全生命周期跨部门协作频繁，传统方式往往存在：

- 台账分散（Excel/纸质）、口径不一，追溯困难；
- 采购与入库记录不一致，资产编码重复或无标准；
- 流转缺少审批/记录，责任人不清；
- 盘点工作量大，结果难以闭环；
- 报修流程不规范，维修进度不可见；
- 缺乏统计分析，管理决策滞后。

EAMS 通过统一数据模型与权限体系，提供标准化业务流程与统计能力，支撑企业资产精细化管理。

### 1.3 术语与缩略语

| 名称 | 说明 |
|---|---|
| EAMS | Enterprise Asset Management System，企业资产管理系统 |
| RBAC | Role-Based Access Control，基于角色的访问控制 |
| 菜单权限 | 控制页面/菜单可见性与访问能力的权限标识 |
| 按钮权限 | 控制页面内操作按钮可见性/可用性的权限标识 |

| 盘点计划 | 指定盘点范围、方式、时间的盘点任务 |
| 报修单 | 资产故障报修的流程工单 |
| 采购单 | 资产采购业务的订单及明细 |

## 2. 总体描述

### 2.1 产品定位
EAMS 面向企业内部资产管理部门、采购部门、财务/审计人员及资产使用部门，提供资产全生命周期管理与统计分析能力，目标是形成“数据统一、流程可控、责任可追、统计可视”的资产管理平台。

### 2.2 业务范围（In Scope / Out of Scope）

#### 2.2.1 范围内（In Scope）
- 用户、角色、部门、菜单/按钮权限（RBAC）管理
- 仪表盘统计与可视化
- 资产分类树管理
- 资产台账管理（查询、维护、查看详情等）
- 资产流转（入库、分配、调拨、归还、报废、送修、维修完成）与历史追踪
- 资产生命周期查询与阶段历史追踪（由业务动作联动写入）
- 盘点计划创建、启动、执行、完成、取消；盘点明细分页查询
- 报修单创建、审批、开始维修、完成维修/拒绝；与资产状态联动
- 采购单创建、查询、取消、详情；入库与批量入库；资金/账单/统计

#### 2.2.2 范围外（Out of Scope）
以下能力在本仓库当前实现中未作为必选交付能力（可在后续迭代纳入）：
- 资产折旧计提与固定资产会计凭证对接
- 与 ERP/财务系统的双向接口对接
- 资产条码/二维码扫码终端（可扩展）
- 移动端/小程序

### 2.3 系统使用者与特征

| 角色（业务） | 典型职责 | 典型操作 |
|---|---|---|
| 系统管理员 | 系统初始化、账号/角色/权限维护 | 用户管理、角色管理、菜单管理、部门管理 |
| 资产管理员 | 资产台账维护、资产流转登记、盘点组织 | 资产管理、流转操作、盘点管理、生命周期查询 |
| 采购管理员 | 采购单管理、采购入库 | 采购管理、待入库、入库/批量入库 |
| 维修人员/管理员 | 报修审批与维修闭环 | 报修审批、开始维修、完成维修 |
| 财务/管理人员 | 统计分析、账单查看 | 财务统计、账单生成/查看、仪表盘 |
| 审计/稽核人员 | 查看记录、追溯与导出 | 查看流转历史、盘点结果、报修记录 |

> 说明：系统采用 RBAC + 权限标识（permission_code）进行控制，具体可分配角色与权限组合以适配不同组织。

### 2.4 运行环境与依赖
- 后端：Java 17、Spring Boot、MySQL 8、Redis（可选/按配置）
- 前端：Vue 3、Vite、Element Plus
- API 前缀：`/api`；默认端口：`9998`
- 鉴权：JWT Bearer Token（有效期以配置为准，当前配置为 1800 秒）

## 3. 业务流程与规则概述

### 3.1 资产主流程（概览）
资产主流程体现“采购→入库→领用/调拨→维修/盘点→报废”的闭环：

```text
采购需求 -> 采购单创建 -> 采购到货 -> 入库(生成资产编码/台账) -> 分配到部门/人员
   |                                                             |
   |                                                             v
   +----------------------------- 盘点/报修/调拨/归还/报废 <------+
```

关键规则（摘要）：
- 资产编码需唯一且可追溯（系统支持自动编号生成策略）。
- 所有影响资产状态的关键动作必须形成“流转记录”和（或）“生命周期记录”。
- 对于敏感操作（如报废、删除等），需具备对应按钮权限/接口权限。

### 3.2 盘点流程（概览）

```text
创建盘点计划 -> 启动盘点 -> 执行盘点(生成/填报明细) -> 完成盘点(汇总差异) -> 归档
          |                          |
          +---- 取消盘点 -------------+
```

### 3.3 报修流程（概览）

```text
提交报修 -> 审批(通过/拒绝) -> 开始维修 -> 完成维修
                     |
                     +-> 拒绝(关闭)
```

## 4. 功能性需求（Functional Requirements）

> 说明：本节以“模块→功能点→规则/输入输出/权限/异常”形式描述。为便于测试与验收，每个需求项提供唯一编号（FR-xxx）。

### 4.1 认证与登录（Auth）

#### FR-001 用户登录
- **描述**：用户使用账号与密码登录，服务端签发 JWT，前端保存并在后续请求中携带。
- **前置条件**：用户账号存在且状态为启用。
- **基本流程**：
  1. 用户输入用户名、密码并提交；
  2. 后端校验账号状态与密码；
  3. 返回 token、用户基础信息；
  4. 前端初始化权限集合与菜单树。
- **业务规则**：
  - token 采用 `Authorization: Bearer <token>` 方式传递；
  - token 过期后需重新登录；
  - 登录失败需提示错误原因（密码错误/账号禁用/账号不存在）。
- **验收点**：正确账号可登录；错误密码不可登录；禁用账号不可登录。

#### FR-002 退出登录
- **描述**：用户主动退出后清除本地 token 与用户信息。
- **验收点**：退出后访问受保护页面会被重定向到登录页。

### 4.2 用户与部门管理（System/User/Department）

#### FR-010 用户查询与分页
- **描述**：支持按用户名/姓名/部门/状态等条件分页查询。
- **数据项**：用户名、昵称/姓名、手机号、部门、角色、状态、创建时间。
- **权限**：`system:user:list`。

#### FR-011 用户新增
- **描述**：管理员新增用户，设置初始密码与部门。
- **校验**：用户名唯一；手机号格式合法；部门存在。
- **权限**：`system:user:add`。

#### FR-012 用户编辑
- **描述**：编辑用户基础信息（昵称、手机号、部门等）。
- **权限**：`system:user:edit`。

#### FR-013 启用/禁用用户
- **描述**：对用户状态进行启用/禁用。
- **权限**：`system:user:status`。

#### FR-014 重置密码
- **描述**：管理员将用户密码重置为默认/随机值，并提示用户下次登录修改。
- **权限**：`system:user:reset`。

#### FR-015 删除用户
- **描述**：删除用户（建议逻辑删除；若物理删除需校验关联）。
- **权限**：`system:user:delete`。

#### FR-016 部门管理（树形）
- **描述**：部门以树形结构维护，支持新增/编辑/删除，支持按部门筛选用户。
- **权限**：
  - 查询：`system:department:list`
  - 新增：`system:department:add`
  - 编辑：`system:department:edit`
  - 删除：`system:department:delete`
- **业务规则**：
  - 删除部门前需校验是否存在子部门或绑定用户；
  - 部门排序可配置。

### 4.3 角色与权限管理（RBAC）

#### FR-020 角色管理
- **描述**：支持角色新增、编辑、删除、启用/禁用。
- **权限**：
  - 列表：`system:role:list`
  - 新增：`system:role:add`
  - 编辑：`system:role:edit`
  - 删除：`system:role:delete`
  - 状态：`system:role:status`

#### FR-021 角色分配菜单/按钮权限
- **描述**：为角色选择菜单树节点（DIR/MENU/BUTTON），保存后即刻生效。
- **权限**：`system:role:permission`。
- **业务规则**：
  - 菜单权限影响左侧菜单可见；
  - 按钮权限影响页面按钮可见（前端 `v-permission` 指令）。

#### FR-022 用户分配角色
- **描述**：为用户绑定一个或多个角色。
- **权限**：`system:user:assign`。

#### FR-023 菜单/权限点管理
- **描述**：维护菜单树（目录/菜单/按钮）与权限标识（permission_code）。
- **权限**：
  - 列表：`system:permission:list`
  - 新增：`system:permission:add`
  - 编辑：`system:permission:edit`
  - 删除：`system:permission:delete`
- **业务规则**：
  - `permission_code` 建议采用 `模块:资源:动作` 命名（如 `asset:info:list`）；
  - MENU 节点需配置 `path` 与组件；BUTTON 节点可仅配置权限标识。

### 4.4 仪表盘统计（Dashboard）

#### FR-030 资产概览统计
- **描述**：展示资产总量、在用/闲置/维修/报废等状态分布，按部门/分类的占比，以及时间趋势（如新增资产趋势）。
- **权限**：`dashboard:view`。
- **验收点**：数据与台账一致；无权限用户不可访问。

### 4.5 资产分类管理（Asset Category）

#### FR-040 分类树维护
- **描述**：维护资产分类树，支持新增根分类/子分类、编辑、删除。
- **权限**：
  - 列表：`asset:category:list`
  - 新增：`asset:category:add`
  - 编辑：`asset:category:edit`
  - 删除：`asset:category:delete`
- **业务规则**：
  - 删除分类需校验是否存在子分类或挂载资产。

### 4.6 资产台账管理（Asset Info）

#### FR-050 资产列表与筛选
- **描述**：支持按资产名称、编码、分类、部门、状态等条件分页查询。
- **权限**：`asset:info:list`。

#### FR-051 资产详情查看
- **描述**：查看资产基础信息、所属分类、使用部门/责任人、当前状态、最近流转记录。
- **权限**：`asset:info:view`。

#### FR-052 资产信息编辑
- **描述**：编辑资产非关键字段（如存放地点、备注等）；关键字段变更需记录。
- **权限**：`asset:info:edit`。
- **业务规则**：
  - 编辑需要审计字段（更新时间、更新人）；
  - 对影响资产归属/状态的变更应通过“流转操作”完成，避免直接改状态。

### 4.7 资产流转管理（Asset Record / Transfer）

#### FR-060 入库（生成资产）
- **描述**：将采购明细或手工入库信息转换为资产台账记录，生成资产编码。
- **权限**：`asset:record:in`。
- **业务规则**：
  - 入库后资产状态为“在库/可分配”；
  - 入库动作产生流转记录；
  - 资产编码遵循编号规则（见技术文档）。

#### FR-061 分配/领用
- **描述**：将资产分配给部门/用户作为责任方。
- **权限**：`asset:record:allocate`。
- **验收点**：分配后责任部门/人更新，流转历史可追溯。

#### FR-062 调拨
- **描述**：资产在部门间调拨，更新归属并形成记录。
- **权限**：`asset:record:transfer`。

#### FR-063 归还
- **描述**：资产归还至库存或指定部门。
- **权限**：`asset:record:return`。

#### FR-064 送修/维修完成
- **描述**：资产送修进入维修态，维修完成恢复可用状态；动作同时可与报修单联动。
- **权限**：
  - 送修：`asset:record:repair`
  - 维修完成：`asset:record:repair-finish`

#### FR-065 报废
- **描述**：资产报废并进入不可用态，禁止后续分配等操作。
- **权限**：`asset:record:scrap`。

#### FR-066 流转历史查询
- **描述**：按资产或时间范围查询流转历史。
- **权限**：`asset:record:history`。

### 4.8 生命周期管理（Lifecycle）

#### FR-070 生命周期查询
- **描述**：查看资产生命周期阶段与历史列表，支持分页。
- **权限**：`lifecycle:list`。
- **业务规则**：
  - 生命周期阶段由入库、分配、调拨、送修、报废等业务动作联动写入；
  - 系统不提供“手工修改生命周期”的入口（避免篡改）。

### 4.9 盘点管理（Inventory）

#### FR-080 创建盘点计划
- **描述**：创建盘点计划，支持“全面盘点/抽样盘点/专项盘点”等类型，设置盘点范围（分类/部门/数量）。
- **权限**：`inventory:create`。
- **业务规则**：
  - 抽样盘点需填写抽样数量；
  - 专项盘点需指定分类或条件；
  - 计划创建后为“未开始”状态。

#### FR-081 启动/取消盘点计划
- **描述**：启动后生成盘点任务，允许在未完成前取消。
- **权限**：
  - 启动：`inventory:start`
  - 取消：`inventory:cancel`

#### FR-082 执行盘点与明细分页
- **描述**：盘点执行过程中，录入或更新盘点明细（资产盘盈/盘亏/正常），支持明细分页查询。
- **权限**：`inventory:execute`。
- **验收点**：明细分页可用；盘点结果可汇总。

#### FR-083 完成盘点
- **描述**：完成后锁定盘点明细并汇总差异，形成可审计结果。
- **权限**：`inventory:complete`。

### 4.10 报修管理（Repair）

#### FR-090 提交报修
- **描述**：对指定资产提交报修单，填写故障描述、图片（可选）、联系人等。
- **权限**：`repair:create`（若实现为公共提交，可按配置放开）。

#### FR-091 报修审批（通过/拒绝）
- **描述**：审批人员对报修单进行处理；通过后可进入维修流程，拒绝则关闭。
- **权限**：`repair:approve`。

#### FR-092 开始维修/完成维修
- **描述**：维修人员更新维修进度，完成后资产恢复可用状态并写入记录。
- **权限**：
  - 开始：`repair:start`
  - 完成：`repair:complete`

#### FR-093 报修详情查询
- **描述**：查看报修单详情与处理历史。
- **权限**：`repair:view`。

### 4.11 采购管理与财务统计（Purchase/Finance）

#### FR-100 采购单管理
- **描述**：创建采购单与明细，支持查询、查看详情、取消。
- **权限**：
  - 列表：`purchase:list`
  - 新建：`purchase:create`
  - 查看：`purchase:view`
  - 取消：`purchase:cancel`

#### FR-101 待入库与入库/批量入库
- **描述**：采购到货后按明细入库，支持批量入库。
- **权限**：`purchase:inbound`（或拆分至更细粒度权限）。

#### FR-102 账单与统计
- **描述**：按月/年生成账单，查看采购资金统计、趋势与分布。
- **权限**：
  - 账单生成：`finance:bill:generate`
  - 账单列表：`finance:bill:list`
  - 统计查看：`finance:statistics:view`

## 5. 数据需求（Data Requirements）

### 5.1 核心数据对象

| 数据对象 | 说明 | 关键字段（示例） |
|---|---|---|
| 用户 | 系统登录与操作主体 | id、username、password、status、department_id |
| 角色 | 权限集合载体 | id、role_code、role_name、status |
| 菜单/权限 | 页面与操作控制点 | id、menu_type、permission_code、path、component |
| 部门 | 组织结构 | id、parent_id、dept_name、sort |
| 资产分类 | 分类树 | id、parent_id、category_name |
| 资产信息 | 资产台账 | id、asset_code、asset_name、category_id、status、dept_id |
| 流转记录 | 状态/归属变更历史 | id、asset_id、record_type、from/to、operator、time |
| 生命周期 | 阶段历史 | id、asset_id、stage、start/end_time、remark |
| 盘点计划 | 盘点任务 | id、type、scope、status、start/end_time |
| 盘点明细 | 盘点结果项 | id、inventory_id、asset_id、result、remark |
| 报修单 | 维修工单 | id、asset_id、status、desc、approve_time |
| 采购单/明细 | 采购业务 | order_id、supplier、amount；detail_id、qty、price |

### 5.2 数据质量要求
- **一致性**：资产状态与最近流转/报修状态保持一致（状态变更必须通过业务动作）。
- **完整性**：资产入库必须具备唯一编码、分类、入库时间等基础字段。
- **可追溯性**：关键动作产生记录并包含操作人、时间、变更前后值（或引用）。
- **安全性**：敏感字段（密码）不可明文存储；日志脱敏。

### 5.3 数据保留与归档
- 资产台账与关键历史数据建议长期保留；
- 盘点与报修记录建议保留不少于（企业自定）年限，用于审计追溯；
- 支持按时间范围检索与归档策略（后续可扩展）。

## 6. 外部接口需求（External Interface Requirements）

### 6.1 用户界面（UI）
- 管理端 Web 界面，兼容主流现代浏览器（Chrome/Edge）。
- 左侧菜单 + 顶部导航 + 主内容区布局；
- 表格列表需提供：分页、筛选、排序（可选）、加载状态、空态提示。

### 6.2 软件接口（API）
- 采用 RESTful 风格 API；
- 统一返回结构（Result<T>）；
- 使用 JWT 鉴权；
- API 文档通过 Swagger 提供（便于联调与验收）。

### 6.3 硬件接口
- 当前版本不依赖特定硬件；若后续扩展扫码枪/盘点机，需在接口层新增对接能力。

## 7. 非功能性需求（Non-Functional Requirements）

### 7.1 性能（NFR-P）
- 常用列表查询接口在正常数据量（如 10 万资产、50 万流转记录）下，应支持分页查询且响应时间可控；
- 盘点明细分页必须稳定可用，避免一次性加载导致前端卡顿。

### 7.2 安全（NFR-S）
- 身份认证：JWT；
- 授权：RBAC + 权限点校验（后端必须强校验，前端仅做展示控制）；
- 密码安全：使用强哈希（BCrypt 等）；
- 接口安全：防止越权、SQL 注入（ORM 参数化）、XSS（前端过滤/后端校验）。

### 7.3 可靠性（NFR-R）
- 核心业务操作（入库、流转、盘点完成、报修审批/完成）需具备事务一致性；
- 异常情况下返回明确错误信息并记录日志。

### 7.4 可用性与易用性（NFR-U）
- 页面操作步骤明确；关键操作（删除、报废、取消等）需二次确认；
- 表单校验提示清晰；
- 权限不足时给出友好提示或自动跳转至可访问页面。

### 7.5 可维护性（NFR-M）
- 后端按 Controller/Service/Mapper 分层，DTO/VO 分离；
- 统一异常处理与统一响应；
- 权限点命名规范化，便于扩展与审计。

## 8. 验收标准（Acceptance Criteria）

### 8.1 功能验收
- 所有 FR 编号功能点可按预期完成，且权限控制符合定义；
- 资产台账、流转、报修、盘点、采购模块数据联动正确；
- 关键数据可追溯（可通过历史记录查询验证）。

### 8.2 非功能验收（示例）
- 安全：无权限用户无法调用受保护接口（验证 401/403）；
- 性能：常用分页接口在指定数据量下响应满足企业要求（可由验收方给出阈值）；
- 可靠性：核心事务操作失败时不产生脏数据。

- 后端服务（可运行包/源码）
