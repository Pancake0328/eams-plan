# 权限管理模块使用说明

> 适用版本：2026-02-08 之后  
> 目标：在前端维护系统菜单与按钮权限，替代手工数据库调整

## 一、入口与权限

- 页面路由：`/permissions`
- 菜单权限：`system:permission:list`
- 操作权限：
  - `system:permission:add`
  - `system:permission:edit`
  - `system:permission:delete`

## 二、功能说明

1. 查看权限树（目录/菜单/按钮）
2. 新增顶级权限与子权限
3. 编辑权限基础信息
4. 删除权限（有子权限时禁止删除）

## 三、字段说明

| 字段 | 说明 |
| --- | --- |
| 菜单名称 | 权限节点名称 |
| 类型 | DIR(目录)、MENU(菜单)、BUTTON(按钮) |
| 权限标识 | `模块:资源:操作`，按钮和菜单建议配置 |
| 路由地址 | DIR/MENU 使用 |
| 组件路径 | MENU 使用 |
| 图标 | DIR/MENU 使用 |
| 排序 | 数字越小越靠前 |
| 状态 | 1-启用，0-禁用 |
| 可见 | 1-显示，0-隐藏 |

## 四、后端接口

| 方法 | 地址 | 权限 |
| --- | --- | --- |
| GET | `/menus/tree` | `system:permission:list` |
| POST | `/menus` | `system:permission:add` |
| PUT | `/menus/{id}` | `system:permission:edit` |
| DELETE | `/menus/{id}` | `system:permission:delete` |

## 五、注意事项

- 删除权限前需先删除其子节点。
- 权限调整后，建议相关用户重新登录以刷新权限缓存。
